# Readwise Ghost Theme Packaging

Use this skill for release, packaging, and CI changes for the Readwise `atilla-rw-custom` Ghost theme.

## Package Flow

- The uploadable Ghost theme ZIP is generated by `grunt compress`.
- The expected ZIP path is `dist/atilla-rw-custom.zip`, derived from `package.json` `name`.
- `dist/` is ignored by git. Do not commit generated ZIP files to the repository.
- GitHub Actions should publish ZIPs as workflow artifacts after merged changes land on `master`.
- Keep repo-only automation, skills, source Sass, source JS, and CI files out of the ZIP.

## Local Commands

1. Install dependencies with `npm ci` if needed.
2. Build compiled assets with `npm run build` or `npx grunt build`.
3. Package the theme with `npm run zip` or `npx grunt compress`.
4. Confirm `dist/atilla-rw-custom.zip` exists.

## CI Expectations

- Pull requests should build and verify committed generated assets.
- Pushes to `master` should build, verify generated assets, create the ZIP, and upload it with `actions/upload-artifact`.
- Avoid workflows that commit ZIPs back to `master`; binary artifacts create noisy history and can trigger extra CI runs.
- Keep GitHub Action permissions minimal. Artifact upload only needs read access to repository contents.

## Validation

- After packaging changes, run `npm run build` and `npm run zip`.
- Inspect the ZIP file list if packaging rules changed.
- Ensure the ZIP contains theme runtime files such as `assets/`, `partials/`, templates, locales, and `package.json`.
- Ensure the ZIP excludes `src/`, `dist/`, `.git/`, `.github/`, `.codex/`, `node_modules/`, `AGENTS.md`, `Gruntfile.js`, and `package-lock.json`.
